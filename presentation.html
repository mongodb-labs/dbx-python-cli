DBX Python CLI

AI-first developer tooling for the MongoDB Python ecosystem.
Repo management · Virtual envs · Testing · Django projects — unified.

The Challenge

MongoDB's Python ecosystem spans many repos, drivers, and integrations. Developers spend too much time on toil instead of code.

Fragmented repos

PyMongo, Motor, Django MongoDB Backend, LangChain — separate repos, each with its own setup ritual.

Virtual env sprawl

Each repo needs its own environment, duplicating dependencies and wasting disk space.

Inconsistent tooling

Every repo has a slightly different dev workflow with no single entry point for common operations.

MongoDB setup overhead

Tests need a running MongoDB instance. Starting the right topology each time is tedious.

The Solution

One CLI to rule the
entire Python ecosystem

DBX Python CLI is installed globally via pipx and gives a single, consistent interface across every MongoDB-related Python project. 

pipx install git+https://github.com/aclark4life/dbx-python-cli.git

dbx clone -g django          # clone the whole django group
dbx install -g django                  # install all deps with uv
dbx test -g django -k "test_find"      # run tests — MongoDB starts automatically

Repository Management

Repos are organised into logical groups. Clone an entire domain with one command. 

[repo.groups.django]
repos = [
  ".../django-mongodb-backend",
  ".../django-mongodb-extensions",
]

[repo.groups.pymongo]
repos = [
  ".../mongo-python-driver",
  ".../motor",
]
  • dbx clone -g django — clone group & run install
  • dbx sync -g pymongo — rebase every repo to upstream
  • dbx list — tree view: ✓ cloned / ○ available
  • dbx switch main -g django — checkout across group
  • dbx status -g langchain — git status for all repos

Global groups are auto-cloned into every other group.
Flat layout option: all repos in one directory.

Smart Virtual Environments

One group-level venv shared by all repos in that group. No duplication; consistent resolution.

uv-powered

Ultra-fast installs. Supports extras, PEP 735 groups, and multi-package monorepos.

Native build steps

Pre-install CMake (libmongocrypt) or Rust/maturin builds for native extensions.

# Create venv for a group
dbx env init -g django

# Install with extras
dbx install -e test,aws

# PEP 735 dependency groups
dbx install --groups dev,docs

# Show all venvs
dbx env list
[repo.groups.django.install_extras]
"django-mongodb-backend" = ["test"]

[repo.groups.pymongo.build_commands]
"mongo-python-driver" = "cmake --build ."
Testing & Projects

MongoDB Auto-Start

No manual mongod setup. Tests and projects get MongoDB automatically via three backends.

runner

Uses npx mongodb-runner. No Docker. Supports standalone, replica set & sharded topologies.

docker

Official MongoDB Docker images. Community or Enterprise. Configurable tag & container name.

atlas-local

MongoDB Atlas Local. Full feature set including Search & Vector Search.

[project.mongodb]
backend  = "runner"     # runner | docker | atlas-local
edition  = "community"  # community | enterprise

[project.mongodb.runner]
topology    = "replset"  # standalone | replset | sharded
secondaries = 2

Django Project Lifecycle

  • dbx project add myapp — scaffold with MongoDB config
  • dbx project install — install project deps
  • dbx project migrate — run Django migrations
  • dbx project su — create superuser
  • dbx project run — start dev server
  • dbx project manage <cmd> — arbitrary commands
  • dbx project list — list all projects
  • dbx project remove myapp — clean up

Quick Start in 4 Steps

# 1. Install globally
pipx install git+https://github.com/\
  aclark4life/dbx-python-cli.git

# 2. Initialise config
dbx config init

# 3. Clone a group + install deps
dbx clone -g pymongo

# 4. Run tests (MongoDB auto-starts)
dbx test -k "test_collection"

Global flags on every command:

FlagEffect
-vVerbose output & tracebacks
-pPipe output through pager
--backendOverride MongoDB backend
--editionOverride MongoDB edition

Requires Python 3.11+. Recommended: 3.13.

Technology Stack

Typer

Modern, type-safe CLI framework. Auto-generated --help for every command.

uv

Ultra-fast package installer & resolver — orders of magnitude faster than pip.

just

Simple make-like command runner. Tight integration via dbx just.

ruff

Fast Python linter & formatter — replaces flake8, isort, black.

pytest + cov

Feature-rich testing with coverage reporting. Custom runners supported.

Design Principles

AI-First

Structured output optimised for AI assistants. All contributions made through AI coding tools.

Standalone Installation

Installed globally via pipx — not tied to any single project. Acts as an orchestrator.

Group-Based Organisation

Operations target a group or the entire ecosystem, not just one repo at a time.

Smart Defaults

Sensible defaults throughout. Most commands work without arguments. Low friction by design.

Flat Command Surface

16 top-level commands. Every feature is one dbx <verb> away.

De-Siloing

Break barriers between teams and repos. One tool, consistent interface, shared conventions.
